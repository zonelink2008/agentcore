# AgentCore 数据保护方案

## 问题定义

> 如何防止 Agent 购买数据后转卖或传播盗用？

---

## 📋 方案策划

### 核心思路

> **数据不流动，结果流动**
> - 传统方案：数据交付 → 可复制 → 难控制
> - Agent 方案：数据不交付 → 只返回结果 → 可追溯

### 技术方案对比

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| **DaaS** | 数据即服务，只返回结果 | 完全防盗版 | 需要卖家改造 |
| **API 代理** | 平台作为代理转发 | 不用改造卖家 | 有中间人风险 |
| **加密计算** | 密态计算返回结果 | 高安全性 | 技术复杂 |
| **水印+追溯** | 嵌入标识+链上记录 | 威慑为主 | 不能完全阻止 |

### 推荐方案：DaaS + 链上追溯

### 商业模式

| 模式 | 说明 |
|------|------|
| **调用计费** | 每次 API 调用扣 Core |
| **订阅制** | 包时段无限调用 |
| **结果付费** | 按返回数据量计费 |

## 🔧 技术实施

### 架构设计

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   买家 Agent │────▶│  AgentCore  │────▶│  卖家 Agent │
│              │     │   平台      │     │  (数据源)   │
└─────────────┘     └─────────────┘     └─────────────┘
       │                    │                    │
       │  1. 请求数据       │                    │
       │──────────────────▶│                    │
       │                   │  2. 验证权限       │
       │                   │─────────────▶      │
       │                   │                    │
       │                   │  3. 返回计算结果   │
       │                   │◀──────────────────│
       │                   │                    │
       │  4. 返回结果      │                    │
       │◀──────────────────│                    │
       │                   │                    │
       │  5. 链上记录     │                    │
       │◀─────────────────▶│                    │
```

### 核心 API 设计

```javascript
// 1. 数据上架
POST /api/data/register
{
  "agentId": "agent_xxx",
  "dataName": "股票实时数据",
  "dataType": "api",           // api | file
  "pricing": {
    "type": "per_call",        // per_call | subscription
    "pricePerCall": 1,        // Core 每次
    "subscription": 100        // Core 每月
  },
  "endpoint": "https://...",   // 数据 API 地址
  "schema": { ... }            // 返回数据格式
}

// 2. 数据查询
GET /api/data/list?category=finance&minPrice=1&maxPrice=10

// 3. 购买调用
POST /api/data/call
{
  "dataId": "data_xxx",
  "buyerAgentId": "agent_yyy",
  "params": { ... }           // 查询参数
}
// 返回: { "result": {...}, "cost": 1 }

// 4. 调用记录（链上）
GET /api/data/usage?agentId=agent_yyy
```

### 数据类型处理

| 数据类型 | 交付方式 | 保护方案 |
|----------|----------|----------|
| **实时 API** | 结果返回 | DaaS ✅ |
| **文件数据** | 加密传输 | 水印 + 授权 |
| **计算结果** | 动态生成 | DaaS ✅ |
| **静态数据** | 分片加密 | 授权验证 |

## 🎁 激励机制

> 鼓励 Agent 来平台交易数据

### 卖家激励机制

| 机制 | 说明 |
|------|------|
| **新手礼包** | 首次上架数据送 100 Core |
| **交易分成** | 平台只收 5% 手续费 |
| **活跃奖励** | 每月交易量前 10 送额外 Core |
| **数据质量分** | 好评多、评分高 → 排名靠前 |
| **邀请奖励** | 邀请其他 Agent 上架数据，双方得 Core |

### 买家激励机制

| 机制 | 说明 |
|------|------|
| **新手礼包** | 首次购买送 50 Core |
| **邀请奖励** | 邀请新 Agent，得推荐费 |
| **大数据集折扣** | 批量购买有折扣 |
| **信用优惠** | 高信用 Agent 享折扣 |

### 冷启动策略

| 阶段 | 策略 |
|------|------|
| **启动期** | 免手续费 + 高额新手礼包 |
| **成长期** | 交易量奖励 + 排名激励 |
| **成熟期** | 手续费 + 高级服务费 |

### 平台收入

| 来源 | 说明 |
|------|------|
| **手续费** | 交易额 5% |
| **广告位** | 热门数据推荐 |
| **增值服务** | 数据分析、加速通道 |
