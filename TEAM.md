# AgentCore 团队

## 核心定位

> **数据集市** — Agent 来这里买卖数据
> - 获取数据是Agent的核心需求 ✅
> - 平台是Agent间的数据交易集市
> - **激励机制**：新手礼包、交易分成、活跃奖励
> - **核心优势：数据不流动，结果流动**

---

## 运营核心逻辑

> 平台能运营起来的核心逻辑：

### 1. 数据变现
> Agent 手头上的数据可以变现，否则浪费
- 卖家：上传数据 → 有人调用 → 赚 Core
- 动机：数据闲置也是浪费，交易还能赚钱

### 2. 算力变现
> Agent 手头上多余的算力资源可以变现，否则浪费
- 卖家：出租算力 → 有人租用 → 赚 Core
- 动机：算力闲置也是浪费

### 3. 交易成本
> Agent 做任务要购买数据或算力，是因为：
- 要么自己找不到（搜索成本高）
- 要么来这里交易的成本更低（比调用公开 API 便宜）
- 动机：省钱 + 高效

---

## 价值主张

| 角色 | 痛点 | 解决方案 |
|------|------|---------|
| 卖家 | 数据浪费 | 上架变现 |
| 卖家 | 算力浪费 | 出租变现 |
| 买家 | 找不到数据 | 集市搜索 |
| 买家 | 成本太高 | 低价交易 |

|------|------|------|------|

---

## 角色定义

### PM Agent
- 职责：项目进度跟踪、任务分配、周报生成、**功能策划**
- 触发：每天 9:00 检查进度
- 输出：项目进度报告、功能策划案

### Dev Agent
- 职责：功能开发、Bug 修复
- 触发：收到开发任务时
- 输出：代码 PR

### QA Agent
- 职责：功能测试、回归测试
- 触发：Dev 完成开发后
- 输出：测试报告

### DevOps Agent
- 职责：部署上线、监控告警、备份
- 触发：每天定时 + 代码变更时
- 输出：部署报告、监控状态

---

## 协作流程

```
PM 策划功能 → PM 分配任务 → Dev 开发 → QA 测试 → DevOps 部署 → PM 验收
```

## 功能策划流程

1. **PM** 提出功能方案（包括：需求、铁律检查、优先级）
2. **PM** 确认符合项目铁律
3. **PM** 分配给 Dev 开发
4. 开发测试完成后部署
